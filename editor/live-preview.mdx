To open live preview, click the **Live Preview** button in the top bar of the editor. Your docs open in a new browser tab, rendered exactly as they will appear when published. The preview updates in real time as you edit — there's no need to save or publish first, and no build is required. Live preview is only available for public documentation sites.

## Live preview vs. preview deployments

Live preview and preview deployments serve different purposes. Use the table below to decide which one fits your situation:

| | Live preview | Preview deployment |
|---|---|---|
| Speed | Instant | Requires a build |
| Access | Local URL | Shareable URL |
| Use when | Reviewing while editing | Sharing with team |
| Availability | While editor is open | While branch exists |

In short: use **live preview** when you want immediate visual feedback as you write. Use a **preview deployment** when you want to share a link with teammates or reviewers so they can see your changes without needing editor access.

<Note>
  Live preview is only available for public documentation sites. If your docs are behind authentication or private, use a preview deployment instead.
</Note>
